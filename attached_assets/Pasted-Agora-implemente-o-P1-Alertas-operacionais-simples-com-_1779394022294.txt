Objetivo:
Criar alertas calculados em tempo real para lembrar o operador de finalizar pendências importantes.

Implementar endpoint:

GET /alerts

Esse endpoint deve retornar contagens como:

{
  "awaitingSettlement": 0,
  "routesInProgress": 0,
  "routesAvailable": 0,
  "readyNotActioned": 0,
  "deliveryWithoutRoute": 0,
  "cashRegisterOpenHours": 0
}

Regras dos alertas:

1. awaitingSettlement
Contar entregas com:
- deliveryStatus = awaiting_settlement
- paymentTiming = on_delivery
- sem payment registrado

2. routesInProgress
Contar rotas com:
- status = in_progress

3. routesAvailable
Contar rotas disponíveis para assumir:
- status = available

4. readyNotActioned
Contar pedidos prontos há mais de 20 minutos que ainda não foram pagos, retirados, entregues ou finalizados.

5. deliveryWithoutRoute
Contar pedidos delivery:
- deliveryStatus = preparing ou ready
- ainda não vinculados a rota ativa

6. cashRegisterOpenHours
Se houver caixa aberto, retornar há quantas horas ele está aberto.
Se não houver caixa aberto, retornar 0.

1. Mostrar badges pequenos no menu lateral:
- Caixa: badge se awaitingSettlement > 0
