2. REGRA CORRETA PARA activeOrdersCount
==================================================

activeOrdersCount deve contar apenas pedidos realmente operacionais da sessão atual.

Contar se:
- orders.createdAt >= operationalStart
- orders.status em open/preparing/ready
- orders.status não é closed/cancelled

Mas excluir delivery quando deliveryStatus estiver em:
- out_for_delivery
- delivered
- awaiting_settlement
- closed
- cancelled

Ou seja:
Pedido aguardando baixa financeira NÃO é pedido ativo operacional.

==================================================
3. REGRA CORRETA PARA readyNotActioned
==================================================

readyNotActioned deve contar apenas pedidos que realmente precisam de ação operacional.

Contar se:
- orders.createdAt >= operationalStart
- orders.status = ready
- coalesce(readyAt, updatedAt) < agora - 20min

Mas excluir se:
- deliveryStatus = out_for_delivery
- deliveryStatus = delivered
- deliveryStatus = awaiting_settlement
- deliveryStatus = closed
- status = closed/cancelled

Pedido aguardando baixa financeira nunca pode aparecer como “pronto sem ação”.

==================================================
4. REGRA CORRETA PARA pendingKitchenCount
==================================================

pendingKitchenCount deve contar a fila real da cozinha, não apenas orders.status.

Usar kitchenTickets como fonte principal.

Contar tickets pendentes se:
- kitchenTickets.status = pending
- pedido relacionado foi criado na sessão atual
- pedido não está closed/cancelled
- pedido não está entregue
- pedido não está awaiting_settlement
- pedido ainda precisa de preparo

Fazer join:
kitchenTickets -> orders

Se um pedido aparece na tela Cozinha como pendente, o badge Cozinha deve aparecer.
