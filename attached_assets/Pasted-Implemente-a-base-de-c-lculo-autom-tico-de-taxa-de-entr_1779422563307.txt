Implemente a base de cálculo automático de taxa de entrega por distância e a preparação para entrada automática de pedidos externos via API.

Não refaça o projeto do zero.
Não mexa no visual geral.
Não altere cores.
Não implemente integração real com iFood agora.
Não use API paga de mapas agora.
Preserve tudo que já funciona:
- pedidos
- cozinha
- pagamento
- caixa
- delivery
- rotas
- baixa financeira
- dashboard
- alertas
- cardápio
- configurações

OBJETIVO:
1. No pedido delivery manual, permitir calcular automaticamente a taxa de entrega com base no CEP/endereço da loja e no CEP/endereço do cliente.
2. Preparar o sistema para receber pedidos externos de iFood, WhatsApp, site ou outra API, criando o pedido automaticamente no PDV com todos os dados originais.

==================================================
PARTE 1 — CONFIGURAÇÕES DE ENTREGA
==================================================

Na página Configurações, adicionar campos para regra de entrega:

- deliveryFeeMode:
  - manual
  - per_km

- deliveryPricePerKm:
  valor cobrado por km

- minimumDeliveryFee:
  taxa mínima de entrega

- maximumDeliveryFee:
  taxa máxima opcional

- storeCep:
  usar o CEP da loja já existente

- storeAddress:
  usar endereço da loja já existente

Regras:

1. Se deliveryFeeMode = manual:
   - a taxa de entrega continua sendo preenchida manualmente no Novo Pedido.

2. Se deliveryFeeMode = per_km:
   - no Novo Pedido delivery, calcular taxa automaticamente com base na distância aproximada.

3. Se houver minimumDeliveryFee:
   - aplicar taxa mínima.

4. Se houver maximumDeliveryFee:
   - não ultrapassar taxa máxima.

Exemplo:
distância = 3 km
preço por km = R$ 2,50
taxa calculada = R$ 7,50

Se taxa mínima = R$ 8,00:
taxa final = R$ 8,00

PARTE 2 — CÁLCULO DE DISTÂNCIA POR CEP PARA MVP
==================================================

Para o MVP, não usar API paga de geolocalização.

Criar uma função de distância aproximada por CEP.

Sugestão:
- normalizar CEP removendo traços e espaços
- usar diferença numérica aproximada entre CEP da loja e CEP do cliente
- converter para uma distância estimada em km de forma simples
- deixar claro no código que isso é uma aproximação para MVP

Criar funções como:

normalizeCep(cep)
estimateDistanceKmFromCep(storeCep, customerCep)
calculateDeliveryFee(distanceKm, settings)

Importante:
Essa estimativa não precisa ser perfeita agora.
Ela serve para dar uma taxa automática inicial.

Se futuramente integrarmos Google Maps, Mapbox ou outra API:
- substituir apenas essa função de cálculo de distância
- sem quebrar o resto do sistema

PARTE 5 — REGRAS PARA PEDIDO EXTERNO
==================================================

Quando um pedido externo chegar:

1. Criar pedido automaticamente no PDV.
2. Salvar source:
- ifood
- whatsapp
- site
- totem
- garcom
- api_externa

3. Salvar externalOrderId.
4. Não criar duplicado se source + externalOrderId já existir.
5. Salvar rawPayload para auditoria.
6. Criar os itens do pedido mesmo que não existam no cardápio local.
7. Salvar nome e preço do item exatamente como veio da API.
8. Recalcular subtotal pelos itens.
9. Se a API mandou delivery.fee:
   - usar essa taxa original
   - deliveryFeeSource = external_api
   - não recalcular por km
10. Se a API não mandou taxa:
   - calcular com a regra interna, se deliveryFeeMode = per_km
   - ou deixar manual/zero, conforme configuração
